# Ravenis Editorial Intelligence Design System

## 1. 视觉主题与氛围

Ravenis 是个人情报编辑台，不是监控驾驶舱。整体使用暖白纸面、近黑正文和克制的靛蓝交互色，让重点判断先于系统状态和数据装饰。

## 2. 色板与角色

- 页面背景：`#f6f5f4`
- 内容表面：`#ffffff`
- 主文字：`rgba(0, 0, 0, 0.88)`
- 次文字：`#615d59`
- 轻分隔：`rgba(0, 0, 0, 0.10)`
- 交互色：`#315b8a`
- 交互浅底：`#eef4fa`
- 成功：`#2f6f52`；提醒：`#a15c20`；错误：`#a33a36`

## 3. 排版规则

- 中文正文与 UI 使用系统字体栈：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Noto Sans SC", sans-serif`。
- 正文 `16px/1.7`，移动端不低于 `15px/1.7`；元数据不低于 `13px/1.5`。
- 字重只用 `400/500/600`；禁止斜体和负字距。
- 数字使用 `font-variant-numeric: tabular-nums`。

## 4. 组件样式

- 常规内容优先使用留白和 `1px solid rgba(0,0,0,.10)` 分隔，不把每段文字都装进卡片。
- 重点摘要可使用 `12px` 圆角白色表面与 4 层低透明度暖灰阴影。
- 输入框和按钮使用 `8px` 圆角；标签只表示类型或状态，使用全圆角。
- 所有链接、按钮和表单控件提供清晰的靛蓝 `focus-visible` 环。

## 5. 布局原则

- 企业微信只承载单条、约 3 分钟可读的编辑式简报；完整记录进入 Pages。
- Pages 首屏顺序固定为：运行摘要、重点、继续观察、筛选、完整列表。
- 桌面内容宽度不超过 `1120px`；正文阅读列控制在 `36em` 左右。
- 使用 `8px` 基准节奏：`8/12/16/24/32/48px`；组内间距必须小于组间间距。

## 6. 深度层级

- Level 0：暖白页面背景，无阴影。
- Level 1：白色表面或 10% 轻边框。
- Level 2：重点摘要使用多层低透明度阴影，不使用悬浮发光。
- Level 3：仅错误或阻塞状态使用明显语义色。

## 7. Do / Don't

- Do：结论前置；标题使用具体主体和变化；空栏目不渲染；链接可深链。
- Do：把来源、首次/最后出现和证据数量作为信任信息。
- Don't：使用“量子祈坛”等需要解释的内部栏目名；重复 `[热]`；把 4000 bytes 当作排版目标。
- Don't：使用紫蓝渐变、玻璃拟态、装饰性中文 Web 字体或常驻加载成功文案。

## 8. 响应式

- `< 768px` 收敛为单列，筛选控件占满可用宽度，触控目标至少 `44px`。
- 移动端首先显示 3 项重点；完整列表每页 `20` 条，避免一次渲染过多内容。
- 页面不得横向滚动；超长标题允许自然换行并使用 `overflow-wrap:anywhere` 兜底。

## 9. Motion 哲学

历史页不依赖动效表达层级。只保留浏览器原生滚动、焦点与即时筛选反馈；不添加入场动画、滚动动画或自动轮播。
